# MVP Spec

## Scope

Build a local web application that guides the inspector through capturing anomaly findings for a roof inspection and generates a polished, editable `.docx` report and searchable `.pdf` from that data. The app runs on the inspector's machine — no internet connection required. One command starts it; Chrome is the window.

## Non-Goals

- Multi-user accounts or auth.
- Cloud hosting.
- Orthomosaic stitching (use 3rd-party software; ingest the result).
- Automatic anomaly detection or radiometric analysis.
- Invoicing or CRM features.
- Full client/property database.

## How It Works

The inspector runs one command to start the local server, then opens the app in Chrome. The workflow is:

1. **Create or open an inspection folder.** Each inspection lives in one folder containing `inspection.yaml` and an `images/` subfolder. The app reads and writes `inspection.yaml` — the user never edits it by hand.
2. **Fill in inspection metadata.** Client, property, date, environmental conditions, technician, and equipment fields are entered via a form.
3. **Mark up nadiral thermal images.** The app displays each thermal image. The user draws polygon outlines around anomalous areas using click-to-close polygons.
4. **Place anomaly IDs on the orthomosaic.** The orthomosaic (generated by 3rd-party software and placed in the inspection folder) is displayed in the app. The user clicks twice to place a labeled marker for each anomaly — one click for position, one to confirm.
5. **Enter findings for each anomaly.** Each anomaly gets location, area (manual entry), severity, observation, recommendation, and associated image(s). Some images may contain more than one anomaly.
6. **Generate the report.** One button produces an editable `.docx` and optionally a searchable `.pdf`. The app confirms the PDF has a real text layer.

## Inputs

Each inspection lives in one folder:

```text
inspection-folder/
  inspection.yaml             ← written and read by the app; never hand-edited
  images/
    aerial-visual.png
    aerial-thermal.png
    orthomosaic.png           ← generated by 3rd-party software, ingested by app
    nadiral-thermal-01.png    ← thermal image(s) for polygon markup
    anomaly-01.png            ← close-up per anomaly (one image may cover multiple)
    anomaly-02.png
```

## Output

The app produces:

- Editable `.docx` report
- Optional searchable `.pdf`
- Validation messages in the app UI

## Required Report Sections

1. Cover
2. Table of contents
3. About this inspection
4. Inspection details
5. Summary of findings
6. Findings table (one row per anomaly: ID, location, area, severity)
7. Roof overview (visual and thermal images)
8. Annotated overview (orthomosaic with anomaly ID markers placed by user)
9. Anomaly detail sections (one per anomaly: polygon-marked thermal image, close-up photo, findings text)
10. Recommended next steps

## Anomaly Data Model

Every anomaly has:

- ID (string, e.g. `"01"`, `"02"` — preserved in report and on orthomosaic)
- Location on roof (text)
- Approximate area in sq ft (manually entered by user)
- Severity (e.g. moderate, severe)
- Observation (text)
- Recommendation (text)
- Associated image(s) — one image may cover more than one anomaly

## Report Scale

- A zero-anomaly report is valid output.
- Reports with 50 or more anomalies must remain navigable via the findings table and TOC.

## Validation Rules

The app should warn or fail clearly when:

- A required field is missing
- An image path does not exist
- Anomaly IDs are duplicated or out of order
- PDF export lacks searchable text
- Certification expiration (if present) is before the inspection date

## Recommended Stack

- **Backend:** Python with Flask or FastAPI — serves the UI and handles report generation
- **Frontend:** HTML/CSS/JavaScript (no heavy framework required for MVP)
- **Image markup:** HTML Canvas API for polygon drawing and orthomosaic marker placement
- **Report generation:** `docxtpl` for DOCX template rendering
- **Data:** `PyYAML` for reading/writing `inspection.yaml`; `pydantic` for validation
- **PDF export:** LibreOffice or Word automation where available
