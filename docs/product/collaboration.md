# Collaboration

## Roles

- David: product direction, implementation coordination, repository owner.
- Nate: domain expert, report user, template owner, acceptance reviewer.

## GitHub Workflow

- Use issues for tasks, decisions, and open questions.
- Use labels to separate `mvp`, `product`, `engineering`, `docs`, `template`, and `question` work.
- Keep each issue small enough to close with one clear change.
- Use pull requests for implementation changes once code exists.

## Decision Rules

Put decisions in writing when they affect:

- report structure
- required anomaly fields
- terminology used in client-facing documents
- template layout
- validation behavior
- scope changes between Phase 1 and Phase 2

## Definition of Done for MVP Work

- The behavior is documented.
- A sample inspection can exercise the behavior.
- Nate can inspect the generated report without reading code.
- Known limitations are recorded in the issue or docs.
